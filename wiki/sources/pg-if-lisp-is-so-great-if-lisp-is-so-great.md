# If Lisp is So Great

## Summary

Paul Graham explores the disconnect between the technical quality of the Lisp programming language and its lack of mainstream popularity, arguing that popularity in software is driven by network effects and management preferences rather than inherent language superiority.

## Key Ideas

- There is no direct correlation between the popularity of a language and its quality.
- Lisp's perceived difficulty and unique syntax act as a barrier to entry for many developers.
- Programming language adoption is subject to network effects, where popularity breeds more libraries and ecosystem support.
- Managers often prefer popular languages to ensure developer replaceability, rather than for technical efficiency.
- The 'slowness' of Lisp is a legacy excuse that no longer holds true against modern benchmarks.

- Managers prefer popular languages to maintain leverage over developers. Evidence: The argument that popular languages make developers more easily replaceable, which is a strategic preference for management.
