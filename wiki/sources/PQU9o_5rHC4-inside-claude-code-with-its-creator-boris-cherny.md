## Summary

Boris Cherny, creator of Claude Code, discusses the development philosophy behind the tool, emphasizing the importance of building for future model capabilities, leveraging latent demand, and maintaining a 'beginner mindset.' He highlights how Claude Code has evolved from a simple terminal-based API wrapper into a powerful agentic tool that has significantly increased developer productivity at Anthropic and beyond.

## Key Ideas

- Build for the model six months from now, not the model of today.
- Follow the principle of 'latent demand': make existing user behaviors easier rather than forcing new ones.
- Avoid over-engineering scaffolding; if a model can do it, wait for the model to improve rather than building complex workarounds.
- The role of the software engineer is shifting toward becoming a 'builder' who manages agents, writes specs, and interacts with users.
- Productivity gains from AI agents are unprecedented, with Anthropic seeing a 150% increase in productivity per engineer.
- The terminal remains a surprisingly effective and elegant form factor for AI-driven development.

- Coding is practically solved for many engineers. Evidence: Boris Cherny reports that he has uninstalled his IDE and writes 100% of his code using Claude Code and Opus.
