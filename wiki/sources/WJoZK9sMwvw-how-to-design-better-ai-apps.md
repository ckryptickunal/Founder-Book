# How To Design Better AI Apps

## Summary

Pete Kumman, YC partner and founder of Optimizely, argues that current AI application design is flawed because it treats AI as a 'horseless carriage'—simply embedding chatbots into legacy software interfaces rather than rethinking workflows. He advocates for 'AI-native' design where users can access and customize system prompts and grant agents access to tools, enabling them to automate repetitive tasks rather than just generating text.

## Key Ideas

- Current AI features in apps like Gmail are often frustrating because they use generic, 'lowest common denominator' system prompts that users cannot edit.
- The 'horseless carriage' analogy explains how developers are just slotting AI into old software paradigms instead of redesigning the user experience.
- AI agents should be treated as programmable assistants that can execute tasks across multiple platforms (Slack, Calendar, Jira) using tools, rather than just chat interfaces.
- Users should have access to the system prompt to define their own tone, preferences, and workflows, shifting the responsibility and power from the developer to the user.
- The future of AI software is moving from simple text generation to autonomous agents capable of performing complex, multi-step workflows.

## Topics

- [[topics/system-prompt-design|System Prompt Design]]: The hidden instructions that define an AI's behavior; the argument is that these should be transparent and user-editable.
- [[topics/ai-native-software|AI-Native Software]]: Designing applications from the ground up to automate repetitive work rather than just adding a chat box to existing software.
- [[topics/agentic-workflows|Agentic Workflows]]: Moving beyond Q&A chatbots to agents that can perform actions like labeling, archiving, and cross-app integration.

## Notable Claims

- Most AI features in existing apps are frustrating because they use generic system prompts designed to avoid liability rather than provide utility. Evidence: The Gmail draft example shows how a generic system prompt produces robotic, impersonal emails that the user would never write.
- Coding agents are currently far ahead of other domain-specific agents because they provide 'bare metal' access to the model. Evidence: Tools like Cursor allow users to interact directly with the model's capabilities without restrictive UI layers.
- The ability to write system prompts will become a standard, intuitive skill for most users, similar to using a computer. Evidence: Prompting is more accessible than traditional file system management and will improve as interfaces evolve.
