# Software for Agents

## Metadata

- Video ID: `515J0QD9p6w`

- Published: 2026-05-02T17:20:18Z

## Summary

Y Combinator argues that the next wave of internet users will be AI agents, necessitating a fundamental shift from human-centric visual software interfaces to machine-readable, agent-first infrastructure.

## Key Ideas

- AI agents are currently forced to use software designed for human interaction, which is inefficient and brittle.
- Future software must prioritize machine-readable interfaces like APIs, MCPs, and CLIs over visual dashboards.
- Agents require comprehensive, programmatic documentation to enable autonomous tool discovery and integration.
- The most significant market opportunity lies in building the foundational software for agents rather than just building the agents themselves.
- Incumbent software companies are unlikely to successfully pivot; new startups built for agents as first-class citizens will lead the market.

## Entities

- [[entities/y-combinator|Y Combinator]] (organization): A startup accelerator and venture capital firm.
- [[entities/mcp|MCP]] (other): Model Context Protocol, a standard for connecting AI models to data and tools.

## Topics

- [[topics/agent-first-software|Agent-First Software]]: The design and development of software applications specifically optimized for autonomous AI agents rather than human users.
- [[topics/machine-readable-interfaces|Machine-Readable Interfaces]]: The shift from GUI-based interaction to API, CLI, and protocol-based communication for software systems.

## Notable Claims

- The next trillion users on the internet will be AI agents. Evidence: The rapid increase in agents browsing the web, conducting research, and managing legacy systems.
- Incumbent software companies will fail to capture the agent market. Evidence: Incumbents are merely bolting on agent support rather than rebuilding software from the ground up for agents as first-class citizens.

## Quotes

> The next trillion users on the internet won't be people. They'll be AI agents.
> Agents are doing it on top of software that was designed for humans clicking buttons in a browser, which is slow, inconsistent, and brittle.
> While everyone else is building agents, the biggest opportunity might be building the software that those agents depend on.
