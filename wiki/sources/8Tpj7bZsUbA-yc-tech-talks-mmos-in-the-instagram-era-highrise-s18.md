## Summary

Jimmy, co-founder and CTO of Pocket Worlds, explains the 'social-first' design philosophy behind their MMO app Highrise. He details a hybrid technical architecture that combines native mobile UI (Swift/Kotlin) with a C++ game engine to create a platform that functions equally as a social network and a game.

## Key Ideas

- Social-first design prioritizes community features like messaging and profiles over traditional game loops.
- Long-term user retention in mobile games is driven by social connections rather than gameplay mechanics.
- Native UI frameworks (iOS/Android) provide a superior user experience compared to game engines like Unity for social apps.
- A hybrid architecture using a shared C++ core with native UI bridges allows for scalable, cross-platform development.
- The 'social-first' architecture is modular, allowing the company to swap game mechanics while keeping the social infrastructure constant.
