# The Roots of Lisp

## Summary

Paul Graham explores the foundational principles of Lisp as established by John McCarthy in 1960, highlighting how McCarthy's mathematical approach to programming created a self-referential language that serves as a blueprint for modern language evolution.

## Key Ideas

- Lisp's core innovation is the use of lists as a unified structure for both code and data.
- John McCarthy's 1960 paper provided a formal, axiomatic foundation for programming similar to Euclid's work in geometry.
- Modern programming languages are increasingly adopting features originally pioneered by Lisp, such as garbage collection and runtime typing.
- Lisp is unique because it can be written in itself, creating a semantic core that allows for diverse dialects.

## Notable Claims

- Modern programming languages are steadily moving toward the Lisp model. Evidence: The trend of adding features like garbage collection and runtime typing to C-style languages.
- Lisp's defining quality is that it can be written in itself. Evidence: The ability to define the language's semantics using its own primitives.
