# Non-technical founder? Learn to hack

## Metadata

- Video ID: `sa-non-technical-founder-learn-to-hack`

- URL: https://www.youtube.com/watch?v=sa-non-technical-founder-learn-to-hack

## Summary

Sam Altman argues that non-technical solo founders should prioritize learning to code rather than spending months searching for a technical cofounder, noting that building a prototype is more effective for progress and attracting talent than relying on business pitches alone.

## Key Ideas

- Non-technical founders often stall their progress by obsessively searching for a technical cofounder.
- Learning to code is a manageable task that provides essential insight for running a software company.
- Building a functional prototype, even if imperfect, is superior to relying on mockups or business plans.
- Finding a cofounder among strangers is rarely successful; existing relationships are preferred.
- Technical skills can be learned in less time than it takes to find the right cofounder.

- Finding a cofounder among strangers rarely works. Evidence: Altman's observation of startup dynamics and the preference for established relationships.
