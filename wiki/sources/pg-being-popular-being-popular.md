# Being Popular

## Metadata

- Video ID: `pg-being-popular`

## Summary

Paul Graham argues that the popularity of a programming language is driven primarily by the preferences of expert hackers rather than technical merit alone. He posits that for a language to succeed, it must be useful for 'throwaway' programs, have powerful libraries, be hackable, and be the scripting language of a popular system. He critiques Common Lisp's decline and outlines the requirements for a 'dream language' that balances brevity, power, and accessibility.

## Key Ideas

- Programming languages are tools for hackers; their popularity depends on whether hackers like them.
- Expert hackers influence the rest of the programming community, making their opinion the most powerful force for long-term adoption.
- A language must be popular to be good, as a large user base drives improvements and evolution.
- The best way to gain initial users is to provide a useful application written in the new language (a 'trojan horse').
- Languages succeed by being the scripting language of a popular system (e.g., C for Unix, Perl for system administration).
- Hackers value brevity and the ability to 'hack' or subvert the language's intended model.
- Big systems often evolve from small, throwaway programs; therefore, a language must be excellent for quick, small tasks.
- Library design is as important as core language design for future programming language success.
- Redesign is critical; software should be constantly improved, but users can hinder this process if they become too attached to existing interfaces.

## Notable Claims

- Programming languages do not become popular based on their merits alone. Evidence: The author notes that legacy software and hype play roles, and that languages must be the scripting language of a popular system to gain traction.
- A language must be popular to be good. Evidence: Popularity brings feedback from users, which leads to necessary improvements and evolution over time.
- Brevity is a primary driver for hacker preference. Evidence: Hackers are 'lazy' and prefer languages that minimize typing and extraneous syntax.
- The best big programs begin as throwaway programs. Evidence: The author cites Perl as an example of a language that evolved from a collection of utilities into a serious tool.
