# Holding a Program in One's Head

## Summary

Paul Graham explores the cognitive state of 'holding a program in one's head,' arguing that the most effective programming occurs when a developer maintains a complete mental model of their code. He outlines eight practices to facilitate this state and critiques how traditional organizational structures often hinder this process by treating programmers as interchangeable parts.

## Key Ideas

- Programming is a creative, intellectual endeavor similar to mathematics, requiring a deep mental model of the problem space.
- The ability to hold a program in one's head is essential for understanding the problem being solved, not just the code itself.
- Organizations often fail at software development because they prioritize standardization and interchangeability over the individual focus required for complex programming.
- Startups can gain a competitive advantage over large companies by fostering environments that allow individual programmers to maintain deep mental focus.
- Specific habits—such as avoiding distractions, working in long stretches, using succinct languages, and rewriting code—are necessary to maintain this mental state.

## Notable Claims

- Organizations are fundamentally designed to prevent the type of individual focus required for great programming. Evidence: Organizations define themselves by making individuals interchangeable, which contradicts the non-parallelizable nature of creative programming ideas.
- Rewriting code is a superior method for understanding a program. Evidence: You must understand a program completely to rewrite it, making the act of rewriting a primary tool for internalizing complex logic.
