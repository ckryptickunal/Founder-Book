## Summary

A Y Combinator-hosted meetup featuring five technical presentations on the cutting edge of AI research, covering topics from protein biology and self-play in LLMs to real-time RAG and formal verification in software engineering.

## Key Ideas

- The 'bitter lesson' suggests that scaling compute and data, rather than human-engineered domain knowledge, is the primary driver of AI progress.
- Self-play in LLMs (asymmetric self-play) allows models to generate their own training tasks, potentially surpassing human-level performance.
- Streaming RAG can significantly reduce latency in voice AI by processing information while the user is still speaking.
- Formal verification (using languages like Lean) is becoming a critical tool for ensuring the correctness of AI-generated code and scientific proofs.
- Managing AI agents like a real-time strategy (RTS) game—focusing on high-parallelism, high-visibility, and rapid course correction—can drastically increase engineering productivity.

## Topics

- [[topics/protein-biology-ai|Protein Biology AI]]: Applying LLM scaling laws to protein sequence modeling to predict structure and function without hand-built features.
- [[topics/self-play-for-llms|Self-Play for LLMs]]: Using asymmetric self-play where a model generates and solves its own tasks to improve beyond human-provided data.
- [[topics/streaming-rag|Streaming RAG]]: Optimizing RAG pipelines for voice agents by retrieving information in parallel with user input to minimize latency.
- [[topics/formal-verification|Formal Verification]]: Using theorem provers like Lean to ensure the correctness of code and mathematical proofs generated by AI.
- [[topics/agentic-workflows|Agentic Workflows]]: Treating AI agent management like an RTS game to maximize productivity through parallelization and rapid feedback loops.

## Notable Claims

- Training on human-generated data limits models to a typical set, making it improbable to reach the full solution space. Evidence: The speaker argues that finite test-time compute cannot overcome the limitations of training on human-biased data.
- Streaming RAG can reduce latency in voice AI by 0.5 to 1.5 seconds. Evidence: Results from a year-old paper comparing streaming RAG against standard RAG on audio-converted benchmarks.
- Lean is a scalable and beautiful language for both theorem proving and functional programming. Evidence: The speaker highlights its use in formalizing neural networks and verifying complex software like flash attention.
- Adopting an 'RTS-style' agentic workflow can increase PR output per engineer by over 60%. Evidence: Internal metrics from Channel AI's engineering team.
