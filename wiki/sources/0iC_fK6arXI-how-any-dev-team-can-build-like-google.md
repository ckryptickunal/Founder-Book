# How any dev team can build like Google

## Summary

Garry Tan interviews Ellie Schleifer, co-founder of Trunk, to discuss how small engineering teams can adopt the high-quality developer experience (DX) tooling typically reserved for tech giants like Google and Facebook. The video demonstrates how Trunk's automation tools, specifically 'Trunk Actions' and 'Merge Queues,' help teams maintain productivity, prevent broken code in the main branch, and reduce time wasted on manual configuration and logical merge conflicts.

## Key Ideas

- Large tech companies maintain high productivity by investing heavily in internal developer tooling.
- Small teams often lose significant time and money due to inefficient workflows, manual dependency management, and broken main branches.
- Trunk Actions automates repetitive tasks (like dependency updates or documentation triggers) to keep developers in a 'flow' state.
- Merge Queues prevent logical conflicts by testing pull requests collectively before they are merged into the main codebase.
- Improving developer experience is a strategic way to maximize the output of an engineering team.

## Topics

- [[topics/developer-experience-dx|Developer Experience (DX)]]: The sum of the tools, processes, and culture that enable engineers to build software efficiently.
- [[topics/merge-queues|Merge Queues]]: A mechanism to ensure that multiple pull requests are tested together to prevent logical conflicts and keep the main branch stable.
- [[topics/automation|Automation]]: Using tools like Trunk Actions to automate manual tasks such as dependency installation and documentation updates.

## Notable Claims

- Small teams lose significant money due to inefficient developer workflows. Evidence: The video calculates that if dozens of developers lose time to simple errors, it can equate to thousands of dollars in wasted runway per day.
- Merge queues are essential for preventing 'broken main' scenarios. Evidence: Demonstrated via a 'Heisenberg' test case where two individually passing PRs cause a failure when merged together.
